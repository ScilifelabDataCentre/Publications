Log in to your account

In order to add publications to this system, you have to be logged in to your account in the system. If you do not have one, request it by contacting the system administrator given in the Contact page.

Add publications by fetching data

This is the recommended procedure for adding publications.

Publication references should be added using the Fetch data for publication page. In that page, do the following:

  1. Enter the Digital Object Identifiers (DOI) and/or PubMed identifiers (PMID) of the publications to be added.
  2. Use the checkboxes to denote which {% module Translate('labels') %} should be applied to the publications. {% if settings['SITE_LABEL_QUALIFIERS'] %} Also select the applicable qualifier. {% end %}
  3. When clicking "Fetch", the system goes to the web services at Crossref and/or PubMed, respectively, to fetch the publication reference data.
  4. This procedure checks whether the DOI or PMID already exists in the database, in order to avoid multiple copies. The {% module Translate('labels') %} {% if settings['SITE_LABEL_QUALIFIERS'] %} and qualifiers {% end %} will be applied to entries that already exist in the database, as if they had been added by the operation.

Add publications manually

Publication references can be added manually by filling in a form on the Add publication manually page. This is for publication that has neither a DOI nor a PMID, often preprints.

Edit a publication

It is possible to edit the details of a publication. However, this is strongly discouraged. It is much more preferable to update contents from PubMed or Crossref.

If a publication as been wrongly marked using a {% module Translate('label') %} that you control, then do not delete the publication. Instead, edit the publication to remove the offending {% module Translate('label') %}.

Delete a publication

A curator may delete any publication. Please note that this is a global operation, affecting the entire database.

This should therefore be done only if the publication is clearly irrelevant for the site as a whole, or if it is a duplicate.
