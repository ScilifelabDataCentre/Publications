The Publications system stores references to publications. A publication reference is an entity in the database, and is called a 'publication' here, for brevity.

There should be only one entry in the database for each publication. The publication contains title, authors, abstract, bibliographic data, and labels.

Adding publications by fetching data

Publication references should be added using the By fetching data page. By giving a list of Digital Object Identifiers (DOI) and/or PubMed identifiers (PMID), the system goes to the web services at Crossref and/or PubMed, respectively, to fetch the publication reference data.

This method checks whether the DOI or PMID already exists in the database, avoiding multiple copies.

This method also allows adding the labels that you as a curator may edit.

Adding publications manually

Publication references can be added manually by filling in a form on the By fetching data page. This is for publication that has neither a DOI nor a PMID, often preprints.

Multiple copies of a publication

Multiple copies of a publication reference may be created by mistake, for example when adding a reference manually that in fact has already been added.

admin only: The admins of the site may view the Duplicates page which uses a heuristic algorithm to identify possible duplicates. One has to edit and/or delete the entries by hand to remove the offending entry.

Account

Only a user with a valid account may add and edit publications. There are two kinds of accounts:

admin
An administrator of the site. Is allowed to do any operation that can be performed via the web interface. In particular, and admin may add accounts, change labels, and execute the publication duplicates procedure.
curator
A curator can add and edit publications. To get a curator account, one has to contact the admin of the site. There is no registration procedure in the system.
